# メモ体系

アウトプット体系の守備範囲外。

## メモ

そのときの感情に左右される。再利用性は低い。自己分析の材料にする。

アウトプットを最優先にしたいため、メモの労力は最小限にしたい。

## 種類


種類|説明
----|----
空想|ばかばかしい空想
要求|これが欲しい
主張|こうすべきだ！

不満|できない

いいね|使える、素敵

ToDo|やること

事故|緊急事態発生（OS、PC壊れた）

* それぞれ１ファイルとする
* 1件あたり80字以内のテキストとする
    * 日時
    * URL（対象1件、言及1件）
    * 画像（複数）
* それぞれはネットワーク関係を構築しうる
    * すべて「やってみる」を葉にできうる

もう少し詳しく。

### 被害

* 期待: (内容) [期待させられた根拠。URLや画像]
* 結果: (内容) [裏切られた結果の証拠URLや画像]
* 今後: (自己防衛手段) [拡散用toot]
